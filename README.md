# BoardGameCollection
Приложение для управления коллекцией настольных игр с возможностью отслеживания игровых сессий и получения рекомендаций.
# Технологии
- Язык: C# 8.0
- UI-Фреймворк: WPF
- Архитектура: MVVM
- База данных: SQLite с Entity Framework Core
- Unit Testing: xUnit с Moq

# Ответы на вопросы
1. Архитектура фильтрации
Для реализации сложной системы фильтрации использован динамический подход с построением LINQ-запросов. Каждый критерий фильтрации (жанр, сложность, количество игроков и т.д.) добавляет соответствующее условие WHERE к базовому запросу. Такой подход обеспечивает гибкость и эффективность без необходимости реализации паттерна спецификаций, так как LINQ-to-Entities предоставляет достаточную мощь для построения динамических запросов.

2. Работа с диапазонами
Для фильтрации по диапазонам (количество игроков, время игры) используется проверка пересечения интервалов:
g.MaxPlayers >= minPlayersValue - максимальное количество игроков в игре должно быть больше или равно минимальному значению фильтра
g.MinPlayers <= maxPlayersValue - минимальное количество игроков в игре должно быть меньше или равно максимальному значению фильтра
Этот алгоритм корректно обрабатывает различные сценарии пересечения диапазонов.

3. Производительность поиска
Для оптимизации производительности:
Используется отложенная загрузка данных (LINQ)
Реализовано кэширование часто используемых данных (статистика, теги)
Для больших объемов данных добавлены индексы в базе данных для полей, по которым происходит фильтрация
Для очень больших коллекций можно реализовать пагинацию результатов

4. Система тегов
Теги реализованы через связь многие-ко-многим между играми и тегами. Теги хранятся в отдельной таблице, что обеспечивает нормализацию данных и эффективный поиск по тегам. Каждый тег имеет уникальное название, а игры могут иметь несколько тегов, что позволяет гибко категоризировать коллекцию.
